# Choru Personal Project Folder Structure (No Ticket IDs)

Use project names for personal work. No ticket-number system is required.

## Location

```text
~/Desktop/choru/choru-notes/1-projects/personal/   # Active personal projects
~/Desktop/choru/choru-notes/4-archive/personal/    # Archived personal projects
```

## Recommended Model

Use two levels:

1. **Project folder** (long-lived)
   - strategy, repo map, notes, status
2. **Plan folder** (short-lived)
   - one initiative/milestone
   - generated by `/superplan`

## Target Layout

```text
1-projects/personal/
└── <project-name>/
    ├── main.md
    ├── repos.md
    ├── plans/
    │   ├── 2026-02-17-mvp-auth/
    │   │   ├── main.md
    │   │   └── task-1-*.md
    │   └── 2026-02-20-api-cleanup/
    │       └── main.md
    └── notes/
        ├── ideas.md
        └── decisions.md
```

## Naming

- Project folder: stable kebab-case (`home-lab`, `content-system`)
- Plan folder: `YYYY-MM-DD-<plan-name>`
- Notes: descriptive kebab-case

## Planning Workflow

1. Create/find project folder
2. Create plan folder under `plans/`
3. Delegate to `/superplan` with `target_directory=<plan-folder>`
4. Link plan from project `main.md`

## Legacy Layout (Before Migration)

Common old pattern:

```text
1-projects/personal/
└── <project-name>/
    ├── main.md
    ├── task-1-*.md
    ├── task-2-*.md
    └── phase-1-*.md
```

## After Migration

```text
1-projects/personal/
└── <project-name>/
    ├── main.md                      # project hub
    ├── plans/
    │   └── 2026-02-17-legacy-import/
    │       ├── main.md              # migrated old plan main
    │       ├── task-1-*.md
    │       └── phase-1-*.md
    └── notes/
```
